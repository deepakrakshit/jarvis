# 🤖 AI Assistance Policy

This project uses AI as a **development accelerator**, not a decision-maker.

AI assistance may be used for:

* code drafting and refactoring
* boilerplate generation
* documentation scaffolding
* repetitive implementation tasks

---

## 👨‍💻 Human Responsibility

All critical engineering decisions remain under human control, including:

* system design and architectural direction
* debugging and root-cause validation
* testing strategy and verification
* security, privacy, and compliance
* production readiness and release approval

---

## 🧭 Guardrails

AI-generated contributions must follow these constraints:

* Deterministic commands must remain reliable and executable
* Factual or time-sensitive queries must prefer live, verifiable sources
* No false claims of memory persistence without actual storage
* No fabricated, cached, or stale system outputs (e.g., speed tests)

---

## 🔍 Review Process

All AI-generated changes are treated as **untrusted proposals**.

Before acceptance:

* behavior must be validated against expected outcomes
* routing, search accuracy, and correction flows must be tested
* edge cases must be considered where applicable

No AI output is assumed correct without verification.

---

## 🚨 Production Policy

Final authority rests with human maintainers.

AI assistance is strictly a **productivity tool** and has no autonomous control over:

* system behavior
* deployment decisions
* production outcomes
